***********************************************************************
                       Версия 4.52 (релиз от 21.05.2003 и старше)
***********************************************************************


    а) Новая версия (2.3) сервера Guardant Net:

        - Клиентские приложения поддерживают широковещательный поиск
          сервера ключа. Это позволяет сократить время регистрации на сервере, 
          а также, в большинстве случаев, избавляет от необходимости дополнительно
          настраивать конфигурационный файл клиента GNCLIENT.INI.
          ВНИМАНИЕ:
          Обновленные библиотечные файлы Guardant Net API будут работать
          только с сервером версии 2.3 и выше

        - Реализован механизм балансировки загрузки сервера: при наличии
          в сети нескольких серверов Guardant Net, удовлетворяющим параметрам
          поиска, клиент будет регистрироваться на том из них, который
          в данный момент менее загружен.

        - В конфигурационный файл клиента GNCLIENT.INI добавлены новые опции, 
          управляющие функцией широковещательного поиска:

          Секция [TIMEOUT]:
          - TO_SEARCH - таймаут широковещательного поиска сервера

          Секция [SERVER]:
          - SEARCH - включение/выключение широковещательного поиска
          - BC_ADDR - адрес широковещания
          - UDP_PORT_CLIENT - номер UDP-порта, через который клиент рассылает
            датаграммы серверам
          - UDP_PORT_SERVER - номер UDP-порта, через который сервер отвечает
            клиенту
          ------------
         Опции  UDP_PORT_CLIENT и UDP_PORT_SERVER также включены
         в конфигурационный файл сервера NNKSRV32.INI.


    б) Новая версия (4.1) монитора Guardant Net:

        - Поддержка отображения всех серверов, запущенных в сети, получение детальной
          информации по каждому из них отдельно.


    в) Новая версия (4.61) драйверов Guardant:
      
        - Повышена надежность работы драйверов  под ОС Windows 2000/XP.

                   Версия 4.1 (релиз от 06.06.2002 и старше)
***********************************************************************

    а) Новая версия (2.0) сервера и API Guardant Net:

	- Поддержка протокола TCP/IP для Win16 и Win32 приложений.

	- Возможность установки нескольких серверов с различными 
	  NetBIOS-именами и IP-адресами.
          В пределах сегмента локальной сети можно запускать несколько 
          серверов Guardant Net. Серверы должны быть запущены на разных 
          компьютерах и обладать уникальными именами. Запуск двух серверов
          (сервера и сервиса, двух сервисов) на одной рабочей станции 
	  невозможен.

	- Усовершенствованный механизм высвобождения "зависших" лицензий.
	  Производится регулярный опрос сервера Guardant Net клиентом 
          (Win16, Win32), как минимум, один раз в 5 минут. Сеанс связи с 
 	  приложением, не отправившим данные в течение 3-х интервалов между 
          опросами, прекращается, если новому клиенту не хватает лицензии.

	- Настройка клиентов на конкретные сервера Guardant Net и протоколы.
	  Копия защищенного приложения (клиент) получает данные о сервере 
	  Guardant Net из файла gnclient.ini. Соответственно этот файл должен 
	  находиться в одной директории с копией защищенного приложения.
	  При отсутствии gnclient.ini всем параметрам сервера Guardant Net 
	  присваиваются значения по умолчанию.
	  Файл gnclient.ini содержит информацию:
	   1. Об используемых сетевых протоколах (секция [PROTOCOLS]). 
	   2. О таймаутах на посылку и прием данных (секция [TIMEOUT]). 
	      При работе на медленных линиях или при большой загрузке 
	      сервера значения таймаутов рекомендуется увеличивать, 
	      чтобы клиент, ожидающий ответ, не отключался по истечении 
	      таймаута.
	   3. О сервере Guardant Net (секция [SERVER]): имя сервера 
	      (по умолчанию NVSK_SRVR), IP адрес сервера и номер порта TCP/IP 
	      (по умолчанию 3182)
	   Аналогичные секции и параметры введены в конфигурационный файл 
	   сервера Guardant Net (nnksrv32.ini).

	Примечания:
	1. DOS-приложения не используют протокол ТСР/ IP, из файла gnclient 
	им требуется информация только о NetBIOS: имя сервера (NB_NAME) 
	и таймауты (TO_SEND, TO_RECEIVE).

	2. В файле nnksrv32.ini значение параметра ServiceInstTimeout 
	(секция [SERVICE]) по умолчанию увеличено до 100 с, чтобы избежать 
	проблем при старте сервера Guardant Net в качестве сервиса 
	Windows NT/2000/XP: ServiceInstTimeout=100

    б) В комплект поставки включен справочник по функциям API в формате 
html-help.

    в) Обновленная утилита программирования ключей с улучшенным редактором 
дампов.

    г) Новая версия драйверов Guardant (4.23)

            ---------------------------------------------
            - Как устанавливать систему на жесткий диск -
            ---------------------------------------------

    ПО Guardant записано в подкаталоге, имя которого  представляет со-
бой дату создания релиза ПО и его версию в формате "ГГ_ММ_ДД.VER".
    Установка начнется  автоматически, как только вы вставите компакт-
диск в дисковод CD-ROM компьютера. Можно либо сразу автоматически уста-
новить ПО Guardant, либо выбрать просмотр компакт-диска CD-R для уста-
новки ПО при помощи  программы SETUP.EXE. Для ее запуска из командной 
строки наберите:

     E:\...\SETUP.EXE

и далее следуйте ее инструкциям.

    В процессе установки  мастер-комплекта  необходимо выбрать тип клю-
чей - Guardant Aptus или Guardant Fidus/Stealth/Net и для ключей выб 
ранного типа задать Общие и Личные коды доступа, а также контрольные 
значения (все они находятся на специальной  листовке, которая выдается 
вместе с мастер-комплектом). Личные коды ключей  Guardant Fidus, Stealth 
и Net  нужно вводить в числовом виде (рекомендуется 16-чная форма). Для
подстановки демо-кодов используется кнопка [ДЕМО].
    Если Вы НЕ ВЫБРАЛИ ключи какого-либо типа, соответствующие им ути-
литы мастер-комплекта  после инсталляции  будут  запускаться  только с
ДЕМОНСТРАЦИОННЫМИ ключами. Тогда, для того чтобы появилась возможность
использовать рабочие ключи этого типа,  необходимо переустановить
мастер-комплект.

    Для  ускорения  процесса установки  можно записать  ПО Guardant  с
дистрибутивного носителя в один каталог вашего жесткого диска и произ-
вести установку дистрибутива из этого каталога.


                     ----------------------------
                     - На что обратить внимание -
                     ----------------------------

    1. ВАЖНО!!! В процессе установки  рабочего  мастер-комплекта будет
создан служебный файл NVCODES.DAT, содержащий информацию о кодах досту-
па к ключу. Этот файл  необходим для работы утилит автоматической защи-
ты и программирования ключа, а также тестовых примеров. Ни защищенные 
приложения, ни утилиты, предназначенные для передачи конечным пользова-
телям, не нуждаются в этом файле. Вы ни в коем случае НЕ ДОЛЖНЫ переда-
вать файл NVCODES.DAT кому бы то ни было!

    2. В некоторых случаях  DOS-программы,  защищенные  автоматической
защитой в режиме работы с кодированными файлами данных, не могут  кор-
ректно обрабатывать такие данные. Типичные примеры - EXE-файлы FoxPro 
for DOS,  созданные в режиме compact, а  также система AutoCAD. Такая 
работа защищенных программ - не следствие ошибки защиты!
Дело в том, что обработкой файлов данных занимается здесь не сама про-
грамма, а некий внешний  программный оверлей, запускаемый  из защищен-
ной программы.  Например, для EXE-файлов FoxPro это файл *.ESL,  а для
системы  AutoCAD - файл ACAD.OVL. Как известно, в этом случае обработ-
ка  кодированных данных  автоматически отключается. Поэтому защищенная
программа  (а на самом деле - запущенный из нее  оверлей)  не может их
правильно  интерпретировать.  Чтобы избежать этой неприятности, нужно
при защите программы дополнительно задать опцию /DEE ("Кодировать фай-
лы данных  в порождаемых процессах"; подробные  объяснения см. в Руко-
водстве пользователя).
    Таким образом, если после  защиты программы  она не может правиль-
но  считать кодированные  файлы данных, - повторите  сеанс защиты, до-
полнительно установив опцию  /DEE. Проблема наверняка будет решена!

                            Ограничения
                            -----------

    Автоматическая защита DOS-программ:
    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    1. Не  рекомендуется  защищать  резидентные  программы (особенно с
привязкой файлов данных),  т.к.  в этом случае программный код вакцины
будет  находиться  в памяти компьютера до момента его выключения,  что
может привести к конфликтам с незащищенным программным обеспечением.

    2. Нельзя  самостоятельно  изменять  время  создания  кодированных
файлов данных DOS-формата! Это могут делать только защищенные програм-
мы, работающие с кодированными  данными.  В противном случае эти файлы
не будут корректно обрабатываться защищенными программами (подробности
см. в Руководстве пользователя).

    3. При использовании утилиты NSDKEY.EXE  невозможно защитить файлы
данных,  имеющие расширение .DAT.  Это сделано для того, чтобы Вы слу-
чайно не закодировали файл вакцины,  имеющий такое же расширение. Если
необходимо  защитить файлы с расширением .DAT,  перед защитой переиме-
нуйте их, а после защиты верните им прежнее расширение.

    Автоматическая защита Windows-программ:
    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    1. Опция /CED  (динамическое  кодирование  задачи)  не  работает в
Windows 95, а с некоторыми  нестандартными задачами - и в других типах
Windows.
